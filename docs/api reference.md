## 1. Overview & Conventions

### Base URL

| Environment | URL |
|---|---|
| Production | `https://api.atlas.example.com` |
| Staging | `https://api-staging.atlas.example.com` |
| Local | `http://localhost:8000` |

### Content Types

- All request bodies are `application/json` unless explicitly multipart.
- All response bodies are `application/json` unless streaming (`text/event-stream`).

### Versioning

API versions are namespaced in the URL path (`/v1/...`). Breaking changes require a new version. Backward-compatible additions are made within an existing version. The header `X-API-Version` is returned on every response.

### Identifiers

All resource IDs are UUIDv7 represented as strings. Sample: `01J0K6N9R4QYZQE9YTV7K1J0XW`.

### Timestamps

All timestamps are ISO 8601 with UTC offset (`2026-05-24T14:32:01.123Z`).

### Idempotency

For POST endpoints that create resources, callers MAY supply an `Idempotency-Key` header (any string ≤ 64 chars). Repeated requests with the same key within 24 hours return the same response without re-creating the resource.

## 2. Authentication

Atlas supports two authentication mechanisms:

### 2.1 User JWT (Browser / Frontend)

Used by the Next.js frontend on behalf of an end user.

```
Authorization: Bearer <jwt>
```

JWTs are obtained via `POST /v1/auth/login` and refreshed via `POST /v1/auth/refresh`. Access token TTL is 1 hour. Refresh tokens are stored as HTTP-only cookies and rotated on every refresh.

Claims:

```json
{
  "sub": "01J0...",
  "tenant_id": "01J0...",
  "role": "admin" | "member",
  "iat": 1716557521,
  "exp": 1716561121
}
```

### 2.2 API Key (MCP / Programmatic)

Used by MCP clients and other programmatic integrations.

```
X-Atlas-API-Key: atlas_sk_<32-base32-chars>
```

API keys are issued via `POST /v1/api-keys` (admin only). The plaintext key is shown exactly once at creation; only `bcrypt(key)` is stored server-side. API keys carry only `tenant_id` (no user); their effective role is `member`.

### 2.3 Required Headers Summary

| Endpoint class | Headers required |
|---|---|
| Public (`/v1/auth/signup`, `/v1/auth/login`, `/healthz`) | None |
| Authenticated user endpoints | `Authorization: Bearer <jwt>` |
| MCP-callable endpoints | `X-Atlas-API-Key: <key>` (or JWT also accepted) |
| Admin-only endpoints | JWT with `role=admin` |

## 3. Errors

All errors follow [RFC 7807 Problem Details](https://datatracker.ietf.org/doc/html/rfc7807):

```json
{
  "type": "https://atlas.example.com/errors/budget-exceeded",
  "title": "Daily budget exceeded",
  "status": 429,
  "detail": "Tenant has consumed $1.00 of $1.00 daily budget.",
  "instance": "/v1/query/stream",
  "trace_id": "tr_01J...",
  "retry_after_seconds": 14400
}
```

### Standard Error Types

| HTTP | `type` slug | Meaning |
|---|---|---|
| 400 | `validation-error` | Request body failed Pydantic validation |
| 401 | `unauthorized` | Missing or invalid credentials |
| 403 | `forbidden` | Authenticated but lacking permission |
| 404 | `not-found` | Resource does not exist or is not visible to caller |
| 409 | `conflict` | Resource state prevents the operation |
| 413 | `payload-too-large` | Upload exceeds 50 MB |
| 415 | `unsupported-media-type` | MIME type not in allowlist |
| 422 | `unprocessable-entity` | Well-formed but semantically invalid |
| 429 | `rate-limited` or `budget-exceeded` | Rate or budget cap hit |
| 500 | `internal-error` | Unexpected server error; check `trace_id` |
| 502 | `upstream-error` | LLM provider returned an error |
| 503 | `service-unavailable` | Circuit open or pool exhausted; `Retry-After` provided |

Validation errors include a `details` array describing each failed field:

```json
{
  "type": "https://atlas.example.com/errors/validation-error",
  "title": "Validation failed",
  "status": 400,
  "details": [
    { "loc": ["body", "query"], "msg": "ensure this value has at least 1 character" }
  ]
}
```

## 4. Pagination

All list endpoints use cursor-based pagination.

### Request

```
GET /v1/documents?limit=20&cursor=eyJpZCI6IjAxSjAuLi4ifQ%3D%3D
```

| Parameter | Type | Default | Constraints |
|---|---|---|---|
| `limit` | int | 20 | 1–100 |
| `cursor` | string (opaque) | — | Returned by previous response |

### Response Envelope

```json
{
  "data": [ /* resources */ ],
  "next_cursor": "eyJpZCI6IjAxSjAuLi4ifQ==",
  "has_more": true
}
```

When `has_more` is `false`, `next_cursor` is `null`.

Cursors are opaque base64-encoded JSON. Do not parse or modify them; pass them back verbatim.

## 5. Rate Limits

| Bucket | Limit | Window |
|---|---|---|
| Per-IP, unauthenticated | 60 req | 1 minute |
| Per-IP, authenticated | 600 req | 1 minute |
| Per-tenant, queries | 60 req | 1 minute |
| Per-tenant, document uploads | 600 req | 1 day |

Every response includes:

```
X-RateLimit-Limit: 600
X-RateLimit-Remaining: 487
X-RateLimit-Reset: 1716557600
```

Exceeded → HTTP 429 with `Retry-After` (seconds).

### 6.1 Auth & Users

#### `POST /v1/auth/signup`

Create a new tenant and its first admin user. Public endpoint.

**Request:**

```json
{
  "tenant_slug": "acme-co",
  "tenant_name": "Acme Co.",
  "email": "alice@acme.co",
  "password": "<≥ 12 chars>"
}
```

**Response 201:**

```json
{
  "tenant": { "id": "01J0...", "slug": "acme-co", "name": "Acme Co." },
  "user": { "id": "01J0...", "email": "alice@acme.co", "role": "admin" },
  "access_token": "eyJhbGciOi...",
  "expires_in": 3600
}
```

Sets `atlas_refresh` HTTP-only cookie.

**Errors:** 400 (validation), 409 (slug or email taken).

---

#### `POST /v1/auth/login`

Exchange credentials for a JWT.

**Request:**

```json
{
  "email": "alice@acme.co",
  "password": "..."
}
```

**Response 200:**

```json
{
  "access_token": "eyJhbGciOi...",
  "expires_in": 3600,
  "user": { "id": "01J0...", "email": "alice@acme.co", "role": "admin" },
  "tenant": { "id": "01J0...", "slug": "acme-co" }
}
```

**Errors:** 401 (invalid credentials), 429 (brute-force protection engaged).

---

#### `POST /v1/auth/refresh`

Exchange the refresh cookie for a new access token. Rotates the refresh token.

**Request:** No body. `atlas_refresh` cookie required.

**Response 200:** Same shape as `/login`.

**Errors:** 401 (missing/expired refresh).

---

#### `POST /v1/auth/logout`

Revoke the current refresh token and clear the cookie.

**Response 204.**

---

#### `GET /v1/me`

Return the current user and tenant. Auth required.

**Response 200:**

```json
{
  "user": { "id": "01J0...", "email": "alice@acme.co", "role": "admin" },
  "tenant": {
    "id": "01J0...",
    "slug": "acme-co",
    "name": "Acme Co.",
    "plan": "free"
  }
}
```

### 6.3 Documents

#### `POST /v1/documents`

Upload a document for ingestion. Auth required.

**Request:** `multipart/form-data`

| Field | Type | Required | Notes |
|---|---|---|---|
| `file` | binary | yes | ≤ 50 MB; MIME in `pdf`, `docx`, `md`, `html`, `txt` |
| `title` | string | no | Defaults to original filename |

**Response 202:**

```json
{
  "id": "01J0...",
  "title": "Q4-2025-Earnings.pdf",
  "status": "pending",
  "byte_size": 1245678,
  "content_hash": "sha256:abc123...",
  "uploaded_at": "2026-05-24T14:32:01.123Z",
  "idempotent": false
}
```

If the same `content_hash` already exists for the tenant, returns 200 with `idempotent: true` and the existing record.

**Errors:** 413 (too large), 415 (bad MIME), 429 (rate limit or budget).

---

List documents in the tenant. Auth required.

**Query parameters:**

| Param | Type | Default | Notes |
|---|---|---|---|
| `limit` | int | 20 | 1–100 |
| `cursor` | string | — | |
| `status` | string | — | `pending`/`processing`/`ready`/`failed` |
| `q` | string | — | Substring match on title |

**Response 200:**

```json
{
  "data": [
    {
      "id": "01J0...",
      "title": "Q4-2025-Earnings.pdf",
      "status": "ready",
      "byte_size": 1245678,
      "uploaded_at": "...",
      "ingested_at": "..."
    }
  ],
  "next_cursor": "...",
  "has_more": false
}
```

---

#### `GET /v1/documents/{id}`

Get document detail + ingestion status. Auth required.

**Response 200:**

```json
{
  "id": "01J0...",
  "title": "Q4-2025-Earnings.pdf",
  "status": "ready",
  "byte_size": 1245678,
  "content_hash": "sha256:...",
  "mime_type": "application/pdf",
  "uploaded_by": "01J0...",
  "uploaded_at": "...",
  "ingested_at": "...",
  "failure_reason": null,
  "chunk_count": 47

```

---

#### `GET /v1/documents/{id}/progress`

Server-Sent Events stream of ingestion progress. Closes when `status` becomes `ready` or `failed`.

**Response:** `text/event-stream`

```
event: progress
data: {"chunks_embedded": 12, "chunks_total": 47, "status": "processing"}

event: progress
data: {"chunks_embedded": 47, "chunks_total": 47, "status": "ready"}

event: done
data: {"document_id": "01J0...", "status": "ready"}
```

---

#### `POST /v1/documents/{id}/reindex`

Re-embed and re-index a document under the current tenant settings. Useful after an embedder switch.

**Response 202:**

```json
{ "id": "01J0...", "status": "pending" }
```

---

#### `DELETE /v1/documents/{id}`

Delete a document and all its chunks, citations, and embeddings. Auth required (admin, or member who uploaded it).

**Response 204.**

**Errors:** 403 (member trying to delete someone else's document), 404.

### 6.5 Query

#### `POST /v1/query`

Non-streaming query. Primarily for tests and MCP. For interactive UI use `GET /v1/query/stream`.

**Request:**

```json
{
  "conversation_id": "01J0...",
  "query": "What was AMD's Q4 2023 revenue?",
  "top_k": 8,
  "backend": "pgvector"
}
```

| Field | Type | Default | Constraints |
|---|---|---|---|
| `conversation_id` | UUID | null | If null, a new conversation is created |
| `query` | string | required | 1–4000 chars |
| `top_k` | int | 8 | 1–20 |
| `backend` | enum | `pgvector` | `pgvector` or `qdrant` (feature-flagged) |

**Response 200:**

```json
{
  "conversation_id": "01J0...",
  "message_id": "01J0...",
  "answer": "AMD reported Q4 2023 revenue of $6.2B [^1], up 10% YoY [^2].",
  "citations": [ { "ordinal": 1, "chunk_id": "...", "document_id": "...", "document_title": "...", "page_number": 3, "excerpt": "..." } ],
  "trace_id": "tr_01J...",
  "verified": true,
  "latency_ms": {
    "rewrite": 240,
    "retrieve": 95,
    "rerank": 380,
    "generate": 720,
    "verify": 150,
    "total": 1585
  },
  "cost_usd": 0.00074,
  "token_usage": { "input": 1840, "output": 122 }
}
```

**Errors:** 429 (budget exceeded), 502 (LLM provider error after fallback chain).

---

#### `GET /v1/query/stream`

Server-Sent Events streaming query. The frontend's primary endpoint.

**Query parameters:**

| Param | Required | Notes |
|---|---|---|
| `conversation_id` | no | If absent, new conversation created |
| `query` | yes | URL-encoded |
| `top_k` | no | Default 8 |
| `backend` | no | Default `pgvector` |

**Response:** `text/event-stream`

```
event: meta
data: {"conversation_id": "01J0...", "message_id": "01J0...", "trace_id": "tr_..."}

event: stage
data: {"stage": "rewrite", "status": "running"}

event: stage
data: {"stage": "rewrite", "status": "done", "ms": 240}

event: stage
data: {"stage": "retrieve", "status": "done", "ms": 95}

event: stage
data: {"stage": "rerank", "status": "done", "ms": 380}

event: token
data: {"text": "AMD "}

event: token
data: {"text": "reported "}

...

event: done
data: {
  "citations": [ /* full list */ ],
  "verified": true,
  "cost_usd": 0.00074,
  "latency_ms": { "total": 1585 },
  "token_usage": { "input": 1840, "output": 122 }
}
```

The `event: done` payload is the canonical record; intermediate events are illustrative.

## 7. Streaming Protocol

### Event Types

All SSE streams in Atlas use named events (`event:` line) so clients can subscribe to event types instead of pattern-matching JSON bodies.

| Event | Where | Payload |
|---|---|---|
| `meta` | Query stream open | `{conversation_id, message_id, trace_id}` |
| `stage` | Query (per node) | `{stage, status, ms?}` |
| `token` | Query (during generate) | `{text}` |
| `done` | Query stream close | Full result envelope |
| `progress` | Ingestion progress | `{chunks_embedded, chunks_total, status}` |
| `error` | Any stream | RFC 7807 error |

### Client Behavior

- Clients SHOULD auto-reconnect on transient network loss using the browser EventSource API.
- The server sets `id:` lines so reconnection sends `Last-Event-ID`; the server resumes from the last delivered event when possible.
- After receiving `done` or `error`, the client MUST close the stream.

### Cancellation

Closing the EventSource cancels the underlying request. The server propagates cancellation to in-flight LLM calls within ~100ms (via asyncio task cancellation), so abandoned streams do not waste tokens.
